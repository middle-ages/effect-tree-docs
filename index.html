    About

    1. The basic immutable generic data structure for encoding generic eager trees with 0-n child nodes per branch, also called Rose Trees and everything you need to efficiently query and operate on them.
    2. Stack-safe folds/unfolds, and a library of operations on trees, from counting to zipping.
    3. Encode/decode trees into indented strings Ã  la YAML, nested arrays, edge lists, path lists, and PrÃ¼fer codes.
    4. Instances for @effect/typeclass with law tests.
    5. Draw themeable trees on the terminal with support for multiline labels.
    6. Testing helpers: Customizable arbitraries and functions to enumerate labeled trees.

    Quick Start

    • Install
    pnpm install effect-tree
    • Import

    Everything can be imported from the top level entry-point.

    import {leaf, type Tree} from 'effect-tree

    const myLeaf: Tree<string> = leaf('hello')
    • Create

    You can create leaves and branches with the functions leaf and branch

    import {Tree, branch, leaf} from 'effect-tree

    const myBranch: Tree<string> = branch(
      'root branch', [
        leaf('hello'),
        leaf('world'),
      ]
    )

    Other ways to create trees:

    1. Building from code Functions like tree, branch, from, of (and alias for leaf), withForest are versions of the basic branch and leaf functions, specialized for composing tree building pipelines.

    2. Decoding from data Decode trees from common encodings: edge lists, nested arrays, indented strings, PrÃ¼fer codes, and a list of tree paths.

    3. Unfold with unfolding functions . Functions like unfoldLevelTree unfold trees using functions for unfolding a single level of tree.

    • Draw
    import {Tree, binaryTree, drawTree, map} from 'effect-tree'
    import {pipe} from 'effect'

    console.log(
      pipe(
        3,
        binaryTree,
        map(s => s.toString()),
        drawTree,
